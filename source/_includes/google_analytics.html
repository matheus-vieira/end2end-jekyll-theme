{%- comment -%} Emits Google Analytics only for production builds. Requirements and behavior: - Build must run with `JEKYLL_ENV=production`. - Configure `ga_measurement_id` and (optionally) `ga_options` in a secondary config file (e.g. `_config.env.vars.yml`) and pass it to the build via `--config _config.yml,_config.env.vars.yml` from your CI workflow. Example workflow step is documented in the README. This include reads `site.ga_measurement_id` and `site.ga_options`, sets a safe default for the options (`{}`), and escapes the JSON so it can be safely embedded inside a single-quoted JS string and parsed with `JSON.parse(...)` in the generated script. {%- endcomment -%} {% if jekyll.environment == "production" %} {%- assign ga_id = site.ga_measurement_id -%} {%- if ga_id and ga_id != "" -%} {%- comment -%} Prepare JSON options: use site.ga_options when provided, otherwise default to an empty object literal. Then escape for safe embedding. {%- endcomment -%} {%- if site.ga_options and site.ga_options != "" -%} {%- comment -%} Accept either a YAML mapping (preferred) or a JSON string in `site.ga_options`. If the value looks like JSON (contains `{`), assume it's already JSON; otherwise, jsonify the mapping. {%- endcomment -%} {%- if site.ga_options contains '{' -%} {%- assign ga_options = site.ga_options -%} {%- else -%} {%- assign ga_options = site.ga_options | jsonify -%} {%- endif -%} {%- else -%} {%- assign ga_options = "{}" -%} {%- endif -%} {%- assign ga_options = ga_options | replace: '\\', '\\\\' | replace: "'", "\\'" | replace: '\n', '\\n' | replace: '\r', '\\r' -%} {%- endif -%} {% endif %}
